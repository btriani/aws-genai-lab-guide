# Labs 12-13 Design Spec: Interactive Playground & Resource Dashboard

**Date:** 2026-04-12
**Repo:** `aws-genai-lab-guide` at `/Users/btriani/aws-genai-lab-guide`
**Adds:** Two new labs (12, 13) with Streamlit apps to the existing 11-lab repo

## Purpose

The existing labs are code-only Jupyter notebooks. Students can't interact with what they built — no UI to chat with models, test RAG, try guardrails, or see what resources exist in their account. These two labs close that gap while teaching exam-relevant skills.

## Lab 12: Interactive GenAI Playground

**Exam Domain:** D2: Building Generative AI Applications (30%)
**Models:** Claude Sonnet 4.5, Haiku 4.5, Llama 3 8B, Mistral 7B
**Est. Cost:** ~$0.50 | **Time:** 75 min
**Dependencies:** setup-resources.py completed. The app auto-creates resources if missing — it creates a Knowledge Base if Lab 05's was cleaned up, and a default Guardrail if Lab 10's was cleaned up. Students can also re-run Labs 05/10 first to have them ready.

### Files

```
labs/12-interactive-playground/
├── 12-interactive-playground.ipynb    # Lab notebook — teaches how to build each tab
└── app.py                             # Final Streamlit app
```

### Notebook Structure

Standard template: Header → Diagram → Dependencies (`%pip install boto3 streamlit`) → Config → Lettered Sections → Closing.

Each lettered section builds one tab of the Streamlit app:

#### Section A — Chat Tab
- Streamlit page with model selector dropdown (Claude Sonnet 4.5, Haiku 4.5, Llama 3 8B, Mistral 7B)
- Temperature slider (0.0-1.0), max_tokens slider
- System prompt text area
- Chat interface with streaming output using `st.write_stream` and `ConverseStream`
- Show token usage and estimated cost per message

#### Section B — RAG Tab
- Text input for questions
- Calls `retrieve_and_generate()` on the Knowledge Base created in Lab 05
- Displays: generated answer, retrieved chunks with relevance scores, source document citations
- Toggle between `RetrieveAndGenerate` (managed) and `Retrieve` + custom prompt (DIY)

#### Section C — Model Compare Tab
- Single prompt input, sends to 2-3 selected models simultaneously
- Side-by-side columns showing each model's response
- Metrics row: response time (seconds), input/output token counts, estimated cost
- Helps students understand model selection trade-offs visually

#### Section D — Guardrails Tab
- Connects to the guardrail created in Lab 10 (or creates a default one if missing)
- Input text area with pre-loaded test prompts (PII, blocked topics, safe prompts)
- Shows: original input, guardrail action (ALLOWED/BLOCKED/ANONYMIZED), filtered output
- Guardrail trace details expandable section

#### Section E — Prompt Lab Tab
- Prompt template selector: zero-shot, few-shot, chain-of-thought
- Template preview showing the full prompt that will be sent
- Variable input fields (e.g., the actual question to insert into the template)
- Output display with token count
- Side-by-side comparison: same question across different prompting techniques

## Lab 13: Resource Dashboard & Cost Monitor

**Exam Domain:** D4: Security, Compliance & Governance (20%)
**Models:** None (focus is infrastructure observability)
**Est. Cost:** ~$0.50 | **Time:** 60 min
**Dependencies:** setup-resources.py completed, best after running several labs

### Files

```
labs/13-resource-dashboard/
├── 13-resource-dashboard.ipynb        # Lab notebook — teaches how to build each panel
└── dashboard.py                       # Final Streamlit dashboard
```

### Notebook Structure

Standard template. Each lettered section builds one dashboard panel:

#### Section A — Resource Inventory
- Query all resources tagged `Project=genai-lab-guide` using `resourcegroupstaggingapi`
- For each resource: display name, type, status, ARN, which lab created it
- Generate direct console links (handle OpenSearch Serverless separately since Resource Groups can't deep-link)
- Show resource relationship map (e.g., "Knowledge Base → uses → OpenSearch collection → stores → embeddings from → S3 whitepapers")

#### Section B — Cost Tracker
- Use `ce` (Cost Explorer) client to get spend by service for the last 7 days
- Bar chart of daily cost using `st.bar_chart`
- Per-service breakdown table: Bedrock, OpenSearch Serverless, S3, Lambda, SageMaker, KMS
- Highlight the #1 cost driver (likely OpenSearch Serverless)
- Show projected monthly cost at current rate

#### Section C — Bedrock Usage
- Query CloudTrail for recent Bedrock API events: `lookup_events(LookupAttributes=[{AttributeKey: EventSource, AttributeValue: bedrock.amazonaws.com}])`
- Display: timestamp, API action (InvokeModel, Converse, CreateAgent, etc.), user, source IP
- If model invocation logging is enabled (from Lab 11): show recent prompts/responses from CloudWatch Logs
- Usage summary: total API calls today, calls by model, calls by API type

#### Section D — OpenSearch Status
- Get collection details: status, endpoint, creation time
- Connect with opensearch-py and list indices: name, document count, size
- Show vector dimensions and index settings
- This solves the "I can't find OpenSearch in the console" problem directly

#### Section E — Health Check
- Run the equivalent of `check-prerequisites.sh` in Python:
  - AWS credentials valid?
  - S3 bucket exists and accessible?
  - IAM roles exist?
  - OpenSearch collection ACTIVE?
  - Bedrock model access enabled?
- Green/red status indicators with `st.success()` / `st.error()`
- For failed checks: show the fix command or console link

### dashboard.py Structure

```python
import streamlit as st
import boto3
import json
from datetime import datetime, timedelta

st.set_page_config(page_title="AWS GenAI Dashboard", layout="wide")

REGION = "us-east-1"
session = boto3.Session(region_name=REGION)

# Sidebar: account info
sts = session.client("sts")
identity = sts.get_caller_identity()
st.sidebar.write(f"Account: {identity['Account']}")
st.sidebar.write(f"Region: {REGION}")

# Tabs
tab_resources, tab_cost, tab_usage, tab_oss, tab_health = st.tabs([
    "Resources", "Cost", "Bedrock Usage", "OpenSearch", "Health Check"
])
```

### Closing Sections

- Key Takeaways (5): Resource tagging strategy, Cost Explorer API, CloudTrail for audit, OpenSearch Serverless monitoring, health checks
- Key Concepts (7): Resource Groups, Cost Explorer, CloudTrail, Invocation Logging, Resource Tagging, CloudWatch Metrics, Health Check
- Exam Preparation (5 Q&A): How to monitor Bedrock usage, cost governance, CloudTrail vs invocation logging
- Cost Breakdown: ~$0.50 (Cost Explorer API calls are $0.01 each, CloudTrail lookup is free)
